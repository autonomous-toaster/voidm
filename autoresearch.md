# Autoresearch: Optimize Tinyllama Prompts for Better Query Expansion Quality

## Objective

Improve the **quality of query expansions** generated by tinyllama-1.1B model in voidm's query expansion module. The goal is to generate more relevant, diverse, and semantically coherent synonym/related-term expansions for search queries without overfitting to benchmark patterns.

**Scope**: Tinyllama prompts only (template strings and few-shot examples). No model changes, no architecture changes, no benchmark tampering.

## Metrics

- **Primary**: `expansion_quality_score` (0.0–1.0, higher is better)
  - Measures relevance, diversity, and semantic coherence of generated expansions
  - Computed by test harness scoring both correctness AND diversity
  - Combines: term relevance (50%), term diversity (25%), format correctness (25%)

- **Secondary**:
  - `latency_ms`: Mean expansion latency (target: <500ms for tinyllama)
  - `parse_success_rate`: Percentage of outputs successfully parsed (target: >95%)
  - `term_count_avg`: Average number of terms per expansion (target: 5-15 terms)

## How to Run

```bash
./autoresearch.sh
```

Outputs `METRIC name=value` format. Example:
```
METRIC expansion_quality_score=0.823
METRIC latency_ms=287
METRIC parse_success_rate=0.97
METRIC term_count_avg=8.2
```

## Implementation Notes

### Measuring Expansion Quality

Quality scoring is computed in `autoresearch.sh` by:
1. Running tinyllama on predefined test queries (20+ diverse queries)
2. For each expansion, scoring on:
   - **Relevance** (50%): Do terms match query intent? (manual baseline or embedding-based)
   - **Diversity** (25%): Do terms span different semantic dimensions?
   - **Format** (25%): Output parseable? Correct structure? No hallucinations?
3. Computing weighted average quality score (0.0–1.0)

### Adding New Prompts

To add a new prompt template:
1. Add `const PROMPT_NAME: &str = r#"..."#;` in `prompts` module
2. Add variant to `TemplateMode` enum
3. Update `get_template()` match statement
4. Modify `QueryExpander` to use new mode (or expose via config)
5. Test with `cargo test --lib query_expansion`

### Debugging

Enable verbose logging with:
```bash
RUST_LOG=debug cargo test --lib query_expansion -- --nocapture
```
