## Current Limitation
Reranking works best on short passages (50-300 chars), but voidm passes full documents (1000+ chars).
Current solution: Truncate to 400 chars (pragmatic compromise).

## Proper Long-Term Solution
Implement passage extraction before reranking:
1. Use BM25 to find sentences containing query terms
2. Extract passage around match (±context sentences)
3. Pass only passage to reranker
4. This properly leverages cross-encoder strengths

## Benefits
- Reranking would work perfectly even on very long documents
- No information loss from truncation
- Aligns with cross-encoder training distribution
- Better scoring on long-form content

## Cost
- Moderate refactoring (1-2 days work)
- Need to implement passage extraction logic
- May need to cache passages for performance
- Integration with apply_reranker()

## Priority
- Low (current solution works adequately)
- Can be deferred to future release
- Leave for when user reports reranking effectiveness issues
